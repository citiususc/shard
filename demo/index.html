SHACL co-construction

Home Rule → Shape Guide → Shapes

Generate SHACL validation shapes from an OWL ontology and its business rules. Choose a workflow: turn a single hand-written rule into one shape, or translate a whole application guide (HTML or PDF) into the full set of shapes, reviewing each one as it is produced.

Workflow 1

Rule → Shape

Write a business rule, pick the ontology target, and generate a single SHACL shape you can edit, validate and accept.

Workflow 2

Guide → Shapes

Upload the full application guide (HTML or PDF). The multi-agent pipeline generates every shape and streams them in for review one by one.

Both workflows share the same ontology, model configuration, prefixes and accepted-shape list. Set them once; they persist as you move between pages. Inference runs through the text2shacl model loader (Databricks by default — no local GPU required).
